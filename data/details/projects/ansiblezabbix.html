Zabbix is an open-source monitoring solution used to track system performance and availability. While there are multiple ways to collect data from client machines, one of the most common methods is using the Zabbix agent.

This project automates the deployment of Zabbix agents across multiple hosts using an Ansible playbook and simplifies the registration of new machines in the Zabbix server via a Python script that interacts with the Zabbix API.


Core Functions

  • Automated installation of the Zabbix agent on multiple machines using an Ansible playbook.
  • Automatic registration of new machines in the Zabbix server using a Python script with the Zabbix API.


Motivation

In my DevOps role, I frequently use Zabbix for monitoring. Since new machines are added to our infrastructure regularly, manually setting them up in Zabbix was time-consuming.

When I joined the team, installing agents and registering them was done manually. To streamline this, I developed a Python script to automate host registration, saving both time and effort.

I open-sourced this project on GitHub so others can benefit from it as well.
You can check the files and how to use it in my Github.
